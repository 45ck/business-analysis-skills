# breakout-structure-designer

## Purpose
Design breakout exercises that help groups generate structured output instead of parallel confusion.

## Trigger this skill when
- A workshop needs breakout sessions but the exercise design is unclear.
- You need parallel stakeholder input without chaos.
- The session must produce comparable outputs from multiple groups.

## Expected inputs
- workshop agenda
- participants
- objective
- artifact target

## Deliverables
- breakout design
- group assignments
- facilitation instructions
- synthesis plan

## Operating procedure
1. Choose groupings based on role mix, topic scope, and desired tension or consensus.
2. Define the breakout task, expected artifact, and timebox.
3. Write simple instructions and output format constraints.
4. Plan how outputs will be synthesized in plenary.
5. Add facilitator checkpoints and recovery plans.

## Quality gates
- Breakouts have a concrete output format.
- Group structure supports the objective.
- There is a synthesis method after the breakouts.

## Handoff targets
- workshop-agenda-builder
- as-is-process-investigator
- to-be-process-designer

## Output style
- Be explicit about uncertainty.
- Prefer structured outputs over loose prose.
- Separate confirmed findings, inferred findings, and open questions.
- Preserve source context where practical.

## Failure modes to avoid
- Do not pretend weak evidence is confirmed fact.
- Do not confuse stakeholder opinion with validated rule or process truth.
- Do not hide missing coverage behind polished wording.
- Do not flatten politically different stakeholders into one generic audience.

## Minimum output skeleton
```md
## Summary
## Findings
## Structured outputs
## Risks or tensions
## Open questions
## Recommended next skill
```
